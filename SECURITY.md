# Security Policy

Report vulnerabilities privately through GitHub Security Advisories. Do not
open public issues containing credentials, private targets, scan evidence, or
working exploitation details.

BurpOllama is intended only for systems you own or have written permission to
test. The maintainers do not authorize testing third-party systems.

Generated evidence, readiness decisions, and marketplace report drafts can
contain private target details. Keep `evidence/`, `reports/`, scope files, and
preflight manifests out of public commits unless you have explicit permission
to disclose them.

For bug bounty work, always verify the current program policy, allowed assets,
rate limits, and prohibited testing rules before running the CLI. Treat
`readiness-check` as a local quality gate, not as proof that a program will
accept a submission.
